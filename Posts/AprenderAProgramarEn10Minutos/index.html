Aprender a programar en solo 10 minutos

Te voy a enseñar a programar desde cero mientras te enseño 2 lenguajes de programacion, te enseñare c# y te enseñare python a la vez, dos lenguajes de programacion altamente utilizados y demandados.

La programación es uno de los Skills mas solicitados a día de hoy, los programadores son de los profesionales mejor pagados de todas las industrias de toda la historia, es por eso que es buena idea aprender a programar, en este post, en conjunto con el video de arriba te enseñare a programar de cero en 2 lenguajes de programación diferentes.

No es necesario que te pongas nervioso, la mayoría de los lenguajes funcionan igual, la sintaxis es diferente pero la lógica es la misma, y la lógica es esta que te mostrare a continuación.

Variables

Lo primero que tienes que aprender son las variables, una variable no es más que una caja en la que poner cosas, nada interesante, estas variables pueden guardar distintos tipos de cosas, números, textos y preguntas de Si o No (Booleanos), puede contener tipos más complejos, pero si entiendes eso estas bien. 


PYHTON
SoyUnaVariableTexto = "Hola Mundo"
SoyUnaVariableNumero = 99
SoyUnaVariableBooleana = True

print(SoyUnaVariableTexto)
print(SoyUnaVariableNumero)
print(SoyUnaVariableBooleana)

C#
string SoyUnaVariableTexto = "Hola Mundo";
int SoyUnaVariableNumero = 99;
bool SoyUnaVariableBooleana = true;

Console.WriteLine(SoyUnaVariableTexto);
Console.WriteLine(SoyUnaVariableNumero);
Console.WriteLine(SoyUnaVariableBooleana);

Operaciones entre variables

Con las variables puedes hacer todo tipo de cosas, como sumarlas entre sí, o convertir un tipo en otro tipo o hacer comparaciones las posibilidades son casi casi que infinitas, ya solo con esto puedes empezar a hacer cosas interesantes, ¿qué tal una calculadora? 


PYHTON
SoyUnaVariableNumero = SoyUnaVariableNumero + 5
SoyUnaVariableTexto = SoyUnaVariableTexto + " Que tal?"
SoyUnaVariableBooleana = (SoyUnaVariableTexto == "Hola Mundo")

C#
SoyUnaVariableNumero = SoyUnaVariableNumero + 5;
SoyUnaVariableTexto = SoyUnaVariableTexto + " Que tal?";
SoyUnaVariableBooleana = (SoyUnaVariableTexto == "Hola Mundo");

Condicionales

Los booleanos son la parte más básica de la computación, son preguntas que se resuelven con un si o con un no, por ejemplo ¿Eres mayor de edad? ¿Ya estas suscrito a mi canal de Youtube? Son preguntas que pueden ser resueltas con el concepto de booleano, podemos preguntarle a la maquina si estas preguntas son si o no con un IF o Condicional


PYTHON
if SoyUnaVariableNumero > 99:
	print("El numero es mayor a 99")
elif SoyUnaVariableNumero < 90:
	print("El numero es menor a 90")
else :
	print("El numero no es menor que 90 ni mayor que 99")

C#
if(SoyUnaVariableNumero > 99)
{
	Console.WriteLine("El numero es mayor a 99");
}
else if(SoyUnaVariableNumero < 90)
{
	Console.WriteLine("El numero es menor a 90");
}
else
{
	Console.WriteLine("El numero no es menor que 90 ni mayor que 99");
}

Si la pregunta tiene una respuesta positiva se ejecuta un trozo de código, de lo contrario se ejecuta otra línea de código.

Bucles

Los bucles son trozos de código que se ejecutan muchas veces dependiendo de una comparación, tenemos dos tipos principales los bucles For y los bucles While. 


PYTHON
#Bucle for
for i in range(0, SoyUnaVariableNumero):
	print("Esta linea se ha ejecutado " + str(i) + " Veces")


#While
while SoyUnaVariableNumero > 100:
	SoyUnaVariableNumero -= 1 
print(SoyUnaVariableNumero)

C#
//Bucle for
for(int i = 0 ; i < SoyUnaVariableNumero; i++)
{
	Console.WriteLine("Esta linea se ha ejecutado " + i + " Veces");
}

//While
while(SoyUnaVariableNumero > 100)
{
	SoyUnaVariableNumero --; //Resta 1
	Console.WriteLine(SoyUnaVariableNumero);
}

La diferencia principal entre estos dos tipos de bloques es que el While puede ser infinito, mientras que el For siempre tiene un límite, con esta estructura lógica ya podemos ir pensando en como funcionan los juegos, un ciclo While que ejecuta funciones como recargar la pantalla y colocar los elementos.

Arreglos

Si seguimos con la analogía en donde una caja es una variable, un arreglo serian un montón de cajas enumeradas, todos los ítems de el arreglo tienen el mismo nombre, pero tienen un identificador o índice el cual sirve para diferenciarlos, cualquier tipo de dato puede convertirse en un arreglo, de esta manera podemos tener por ejemplo un arreglo con todos los juegos que has adquirido en Steam.


PYTHON
MiPrimerArreglo = []
MiPrimerArreglo.append("Hola")
MiPrimerArreglo.append("Mundo")
MiPrimerArreglo.append("Como")
MiPrimerArreglo.append("Estas")
for i in range(0, len(MiPrimerArreglo)):
	print(MiPrimerArreglo[i])

C#
string[] MiPrimerArreglo = new string[4];
MiPrimerArreglo[0] = "Hola";
MiPrimerArreglo[1] = "Mundo";
MiPrimerArreglo[2] = "Como";
MiPrimerArreglo[3] = "Estas";
for(int i = 0 ; i < MiPrimerArreglo.Length; i++)
{
	Console.WriteLine(MiPrimerArreglo[i]);
}

Hay que tener cuidado ya que en Python los arreglos son dinámicos, ósea pueden contener tantos ítems como necesitemos, mientras que en C# son estáticos, primero hay que definir que tan grande es el arreglo. También es necesario saber que los índices empiezan en cero, por lo que el ultimo ítem siempre tendrá como valor el tamaño del arreglo menos uno.

Funciones

Muchas veces se da el caso de que tenemos que escribir muchas veces un trozo de código, el trabajo de las funciones es ponerle un nombre a uno de esos fragmentos de códigos para ejecutarlos en cualquier momento, las funciones pueden tener parámetros lo que vuelve mas dinamico el código, puedes verlo claramente en el ejemplo y además pueden retornar valores, eso es genial por que las se vuelven aún más y más dinámicas.


PYTHON
def MiPrimeraFuncion():
	print("Hola Mundo")
	
def MiSegundaFuncion(parametro):
	print("Hola Mundo" + parametro)

def MiTerceraFuncion():
	return "Hola Mundo"
	

MiPrimeraFuncion()
MiSegundaFuncion("Que tal")
print(MiTerceraFuncion())

C#
MiPrimeraFuncion();
MiSegundaFuncion("Que tal");
Console.WriteLine(MiTerceraFuncion());

void MiPrimeraFuncion()
{
	Console.WriteLine("Hola Mundo");
}
void MiSegundaFuncion(string parametro)
{
	Console.WriteLine("Hola Mundo" + parametro);
}
string MiTerceraFuncion()
{
	return "Hola Mundo";
}

Hay que tener ciertas consideraciones, en Python la función debe estar encima de la línea donde es llamada, en C# esto es irrelevante, sin embargo, en C# las funciones siempre deben ser llamadas dentro de otras funciones, usualmente será la función Main, lo veras luego cuando estés jugando con él código.

¿Ya estás viendo las relaciones no? Así es Console.WriteLine y Print, son funciones provenientes de el propio lenguaje, es interesante como hasta el propio lenguaje esta basado en estas simples reglas que te muestro aquí.

Programacion orientada a Objetos

Hasta ahora todo ha sido fácil y bonito, eso se termina aquí, Jajaja nope, esto de los objetos es lo que mas se le dificulta a las personas de entender, pero te lo explicare muy fácil, un objeto es solo un trozo de código en donde hay datos (Variables) y funciones, ¡es todo!

Imagina que tienes el objeto perro, ese perro tiene ciertos datos como su nombre o su edad y también algunas funciones, el perro puede comer o puede ladrar, el perro es un objeto, podemos también tener otro objeto como una persona, la persona tiene nombre, edad, puede también tener un empleo y algunas cosas más, esta persona tiene acciones, puede estudiar, trabajar, viajar, etc. ¡Las personas también son objetos!

¿Si lo miras así es muy fácil no? Este tipo de ideas o abstracciones (Variable, Función, Objetos) están pensadas para imitar la realidad que nos rodea, eso tiene dos ventajas, es mas intuitivo para los programadores y es mas fácil reutilizar el código.


PYTHON
class Perro:
	def __init__(self, nombre):  
	self.nombre = nombre;
	print("Hola soy " + self.nombre + " Soy un perro");
	
	def Comer(self, comida):  
	print(self.nombre + " Esta comiendo " + comida);
	
	def Ladrar(self):  
	print(self.nombre + ": Woof Woof");
lulu = Perro("Lulu")
lulu.Comer("Croquetas")
lulu.Ladrar()

C#
public class Perro
{
	string nombre;
	public Perro(string nombre)
	{
		this.nombre = nombre;
		console.WriteLine("Hola soy " + nombre + " Soy un perro");
	}
	public void Comer(string comida)
	{
		console.WriteLine(nombre + " Esta comiendo " + comida);
	}
	public void Ladrar()
	{
		console.WriteLine(nombre + ": Woof Woof");
	}
}

class MainClass {
	public static void Main (string[] args){
		Perro lulu = new Perro("Lulu");
		lulu.Comer("Croquetas");
		lulu.Ladrar();
	}
}

Ya esto es lo mas alto de la programación, no podemos abstraernos más, sin embargo, hay algo que si podemos hacer para mejorar aun mas nuestros conocimientos, los objetos pueden tener hijos… no literalmente, pero si podemos llegar a decir que un objeto hereda de otro sus propiedades y además tiene propiedades nuevas, por ejemplo ¿habíamos dicho que tanto las personas como los perros tienen nombre y edad y además comen no? Pues podemos asumir que son propiedades que heredan estos dos objetos de una clase superior, la clase Animal, los animales, todos los animales tienen nombre, edad y comen, pero los perros ladran y las personas piensan.

	
PYTHON
class Animal:
	def __init__(self, nombre):  
		self.nombre = nombre;
		print("Hola soy " + self.nombre);

	def Comer(self, comida):  
		print(self.nombre + " Esta comiendo " + comida);

class Perro(Animal):
	def Ladrar(self):  
		print(self.nombre + ": Woof Woof");
	
class Humano(Animal):
	def Pensar(self):  
		print(self.nombre + ": Esta pensando...");

C#
public class Animal
{
	string nombre;
	public Animal(string Animal)
	{
		this.nombre = nombre;
		console.WriteLine("Hola soy " + nombre);
	}
	public void Comer(string comida)
	{
		console.WriteLine(nombre + " Esta comiendo " + comida);
	}
}
public class Humano : Animal
{
	public void Pensar()
	{
		console.WriteLine(nombre + ": Esta pensando");
	}
}
public class Perro : Animal
{
	public void Ladrar()
	{
		console.WriteLine(nombre + ": Woof Woof");
	}
}
